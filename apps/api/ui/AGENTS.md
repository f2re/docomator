# UI agent instructions

## Цель

Интерфейс Docomator должен сопровождать пользователя, а не заставлять его угадывать состояние системы. Источник требований — `docs/UX_UI_SPECIFICATION.md` и раздел UX в `docs/REQUIREMENTS.md`.

## Обязательные правила

- Для каждой сетевой и длительной операции реализуйте понятные состояния загрузки, успеха, ошибки и ограниченной работы.
- Не используйте безымянный бесконечный индикатор ожидания или вымышленный процент.
- Ошибка отвечает на вопросы: что произошло, сохранены ли данные, что делать дальше, какой идентификатор операции.
- Пустой экран объясняет причину и предлагает одно основное действие.
- Недоступная функция честно помечается как запланированная; не имитируйте готовность.
- Под нетривиальным полем должна быть короткая вторичная подсказка серым текстом с достаточным контрастом.
- Эмодзи и цвет не заменяют текстовую подпись, состояние и `aria-label`.
- Минимальная зона касания — 44 × 44 CSS px.
- Поддерживайте управление клавиатурой, видимый фокус, голосовые объявления, ширину 320 пикселей, тёмную тему и уменьшение анимации.
- Не добавляйте внешние сети доставки содержимого, шрифты, аналитику или запросы к внешним доменам во время работы.
- При серверной ошибке форма сохраняет введённые значения.

- Все пользовательские названия, подсказки, состояния и ошибки пишутся по-русски; машинные ключи скрываются либо сопровождаются объяснением.
- Используйте единый словарь отображения для ролей, состояний, типов значений и режимов результата.
- Перед завершением изменения запускайте `npm run check:language`.

## Definition of Done

UI-изменение готово только после проверки применимых состояний, мобильной компоновки, клавиатуры, фокуса, текста восстановления, offline impact и обновления roadmap/ТЗ.
