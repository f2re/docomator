Универсальные данные

База знаний

Здесь хранятся не документы, а переиспользуемые сведения. Система будет подставлять их в разные шаблоны и показывать источник каждого значения.

ⓘ

Тип сущности описывает класс объектов. Например, тип «Человек» объединяет сотрудников, авторов и получателей.

📄 Этап M2–M3

Шаблоны появятся следующим крупным этапом

Сначала система будет безопасно проверять DOCX/XLSX, показывать совместимость и структуру, затем позволит отмечать вариативные поля. Сейчас этот раздел честно недоступен — ваши файлы не будут приниматься до готовности защитных проверок.

Что будет видноЗагрузка → проверка → найденные поля → тестовое заполнение → активация.
Что делать сейчасПодготовить примеры документов и заполнить базу знаний.
✨ Этап M4

Создание документов станет пошаговым

Сервис покажет выбранный шаблон, найденные данные, недостающие поля, предварительный результат и финальное подтверждение. Каждый длительный этап будет сопровождаться понятным статусом и безопасным вариантом продолжения.

1. Выбор→2. Данные→3. Проверка→4. Готово
⏱️ Этап M7

Автоматизации будут объяснять каждый запуск

Для расписаний и событий интерфейс покажет, что запустило правило, какие данные использованы, на каком шаге находится документ, когда будет следующая попытка и требуется ли участие оператора.

Без скрытых действийИстория запусков, идемпотентность и аудит будут доступны из карточки правила.
Без молчаливых ошибокНедостающие данные создадут задачу проверки, а не неполный документ.
