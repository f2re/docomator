# Учебные примеры Docomator

Все имена и сведения в этом каталоге вымышлены. Файлы не содержат рабочих персональных данных и не импортируются, не подключаются и не активируются автоматически.

## Быстрый сценарий

1. На экране «Сотрудники» откройте массовый импорт и выберите `data/employees.csv`.
2. Укажите «ФИО» как отображаемое имя, «Табельный номер» как колонку повторного сопоставления, а оставшиеся колонки создайте как понятные поля карточки.
3. В разделе «Шаблоны» подключите `templates/personal-card.docx`.
4. Во втором столбце карточки сопоставьте четыре значения: ФИО, должность, подразделение и дату приёма.
5. Для сводного выпуска подключите `templates/team-register.xlsx`, сопоставьте ячейки `A4:D4` и явно выберите эту использованную строку как повторяемый диапазон.
6. Сравните результат с файлами из `expected/`. Они показывают только ожидаемый вид и не являются эталоном побайтового результата.

## Состав

- `data/employees.csv` — три вымышленных сотрудника в UTF-8 и с заголовками на русском языке;
- `templates/personal-card.docx` — простой личный документ с четырьмя отдельными местами заполнения;
- `templates/team-register.xlsx` — сводная таблица с одной строкой-образцом без формул;
- `expected/` — заполненные варианты для визуального сравнения.

## Происхождение и ограничения

Эти учебные OOXML-файлы детерминированно собраны из фиксированных локальных частей сценарием `scripts/ci/generate-examples.mjs`. Проверка проекта требует точного SHA-256, закрытого списка частей, отсутствия макросов, формул, внешних связей, полей Word и внедрённых объектов. Затем производственные парсеры выполняют полный intake/structure, компиляцию, заполнение и обратное чтение DOCX/XLSX.

Файлы не выдаются за документы, сохранённые конкретной версией LibreOffice или Microsoft Office. Совместимость с ними подтверждается только отдельной матрицей и фактическим прогоном на целевом Linux-стенде. До такого прогона это безопасный воспроизводимый учебный набор, а не закрытие P6.

Для осознанного обновления сначала измените локальные литералы генератора, затем выполните:

```bash
npm run build
npm run generate:examples
npm run check:examples
```

`manifest.sha256` создаётся вместе с файлами и проверяется также внутри автономного комплекта.
