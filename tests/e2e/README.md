# Локальный E2E-контур Docomator

Сценарии проверяют пользовательский путь без изменения production UI/API и без обращения к внешней сети. Ответы локального API подменяются внутри браузерного контекста Playwright; статические файлы интерфейса загружаются с запущенного локального Docomator.

Покрыты:

- основная навигация и отсутствие горизонтального переполнения на ширинах 320, 768 и 1440 px;
- увеличение текста до 200% на ширинах 320 и 768 px без горизонтального переполнения;
- добавление сотрудника и общего поля по понятному названию;
- импорт CSV без ввода или отправки технических ключей;
- полный мастер DOCX и XLSX: документ, поле, пробное заполнение, PDF-предпросмотр, активация и готовый каталог;
- продолжение настройки из сохранённого исходника без повторного выбора файла, восстановление черновика и активного шаблона после перезагрузки, изоляция при смене раздела и ошибка сервера с сохранением формы и идентификатором операции;
- ровно одна видимая панель мастера, не более одной основной кнопки на шаге и touch target не меньше 44 × 44 px;
- выпуск трёх личных карточек, загрузка ZIP-комплекта и переход в общее хранилище результатов;
- светлая и тёмная темы, клавиатурный фокус и `prefers-reduced-motion`;
- машинно-выявляемые нарушения WCAG на пяти ключевых экранах и в раскрытой форме сотрудника: 320 px в светлой теме и 1440 px в тёмной, без исключённых axe-правил;
- явные PNG-вложения светлой и тёмной темы на каждой из трёх ширин экрана.

Автоматическая fixture-проверка завершает любой сценарий ошибкой, если UI отправил HTTP(S)-запрос на origin, отличный от настроенного `DOCOMATOR_E2E_BASE_URL`.

Axe-аудит не доказывает полное соответствие WCAG 2.2 AA и не заменяет ручную проверку экранным диктором, полного клавиатурного маршрута, порядка и возврата фокуса и пользовательскую приёмку.

## Предварительные условия

1. Собрать проект и запустить API/UI локально, например на `http://127.0.0.1:18080`.
2. Предоставить браузер Chromium в утверждённом офлайн-наборе. Пакет `@playwright/test` зафиксирован в `package-lock.json`, но команда установки браузеров намеренно не запускается автоматически.

## Запуск

```bash
DOCOMATOR_E2E_BASE_URL=http://127.0.0.1:18080 \
  npm run test:e2e
```

Один размер экрана:

```bash
npm run test:e2e -- --project chromium-320
```

Только машинный аудит доступности:

```bash
npm run test:e2e:a11y
```

HTML-отчёт и диагностические материалы пишутся только в `.tmp/playwright-report` и `.tmp/playwright-results`. Каждый полный прогон также создаёт `.tmp/playwright-report.json` с версией точного inventory из 81 выполнения; если выполнялась axe-матрица, отдельный агрегатор атомарно создаёт `.tmp/axe-report.json` с полными результатами 12 обязательных проверок. Трассировка и диагностический снимок сохраняются при падении сценария; визуальный сценарий дополнительно прикладывает успешные PNG для обеих тем и всех размеров экрана.

Эти JSON-файлы не считаются ручным актом сами по себе. На каноническом Linux-стенде их нужно передать fail-closed сборщику из [протокола UX-приёмки](../../docs/UX_ACCEPTANCE_PROTOCOL.md), который проверит точный inventory, скопирует байты под content-addressed именами и создаст отдельный производный акт с заполненным `automationEvidence`, не изменяя входной файл. Все axe-результаты `incomplete` превращаются в явные обязательные строки ручного разбора, привязанные к SHA-256 отчёта, а не считаются автоматически успешными.

## Визуальные эталоны

Пиксельные эталоны создаются и обновляются только на утверждённом стенде Ubuntu 24.04 или целевой Astra Linux с зафиксированным Chromium. Локальные снимки macOS остаются диагностическими и не считаются эталоном.

Проверка уже утверждённых снимков:

```bash
DOCOMATOR_E2E_BASE_URL=http://127.0.0.1:18080 \
  npm run test:e2e:visual
```

Осознанное обновление снимков после визуальной приёмки:

```bash
DOCOMATOR_E2E_BASE_URL=http://127.0.0.1:18080 \
  npm run test:e2e:visual:update
```

Эталоны размещаются в `tests/e2e/snapshots/`. Перед обновлением нужно проверить все шесть вариантов: 320, 768 и 1440 пикселей в светлой и тёмной темах. Обычный `npm run test:e2e` не меняет утверждённые файлы.

Без запуска браузера можно выполнить статическую проверку файлов:

```bash
node --check tests/e2e/playwright.config.mjs
node --check tests/e2e/reporters/axe-json-reporter.mjs
node --check tests/e2e/fixtures/test.mjs
node --check tests/e2e/fixtures/docomator-api.mjs
node --check tests/e2e/pages/docomator-page.mjs
node --check tests/e2e/accessibility-audit.spec.mjs
node --check tests/e2e/navigation-and-accessibility.spec.mjs
node --check tests/e2e/bulk-import.spec.mjs
node --check tests/e2e/employee-card.spec.mjs
node --check tests/e2e/template-and-generation.spec.mjs
node --check tests/e2e/visual-artifacts.spec.mjs
```
