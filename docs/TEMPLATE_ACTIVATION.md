# 👁️ Предварительный просмотр и активация шаблонов

Статус: **сохраняемое создание PDF, явное подтверждение и единый каталог одно- и многополевых версий реализованы**
Последнее обновление: **2026-07-16**

## Назначение

Подсистема завершает ручной путь подготовки проверенной версии шаблона:

```text
одно- или многополевая проверенная версия
        ↓
единый неизменяемый кандидат на выпуск
        ↓
сохраняемая задача предварительного просмотра
        ↓
LibreOffice с отдельным временным профилем
        ↓
проверенный PDF с SHA-256
        ↓
просмотр пользователем
        ↓
явное подтверждение
        ↓
неизменяемый активный выпуск
        ↓
единый каталог текущего пространства
```

Предварительный просмотр не является основным способом изменения DOCX/XLSX. Технические привязки и пробные значения создаёт детерминированный компилятор; LibreOffice только формирует проверочную PDF-копию.

## Единый кандидат на выпуск

Одно- и многополевые проверенные версии остаются в своих совместимых таблицах, но перед предварительным просмотром регистрируются в одном слое кандидатов на выпуск.

Кандидат содержит:

- вид исходной версии: `single` или `multi`;
- пространство и черновик;
- номер исходной проверенной версии;
- формат DOCX/XLSX;
- техническую и пробно заполненную копии;
- SHA-256 обеих копий;
- число полей;
- упорядоченную неизменяемую строку каждого поля;
- исходную и техническую привязки каждого поля;
- версионированный декларативный форматтер каждого поля.

Миграция переносит прежние одно-полевые предварительные просмотры, активные версии и текущие указатели в единый слой выпусков. Исходные записи не изменяются.

## Сохраняемая операция

Запрос предварительного просмотра хранится в SQLite и связан с заданием очереди типа:

```text
template.preview
```

Запись содержит:

- пространство;
- кандидата на выпуск;
- вид и номер исходной проверенной версии;
- число полей;
- идентификатор задания;
- номер явной попытки;
- состояние;
- PDF и его SHA-256;
- сведения о преобразователе;
- понятную ошибку;
- автора, идентификатор операции и время.

Состояния:

| Машинное состояние | Пользовательское отображение |
|---|---|
| `pending` + задание ожидает | Задача поставлена в очередь |
| `pending` + задание выполняется | LibreOffice создаёт PDF |
| `ready` | Предварительный просмотр готов |
| `failed` | Предварительный просмотр не создан |

Ожидающий или готовый запрос не создаёт дубликат задания. После ошибки пользователь явно запускает новую попытку; она получает новый номер и новый ключ защиты от повторов.

## Ограниченный запуск LibreOffice

Фоновый обработчик запускает LibreOffice:

- без оболочки;
- без графического интерфейса;
- с отдельным временным профилем пользователя;
- с отдельными каталогами `HOME`, `XDG_*` и `TMPDIR`;
- с внешним ограничением времени;
- с принудительным завершением группы процессов при отмене или превышении времени;
- с ограничением входного файла, PDF и захватываемого вывода процесса;
- без передачи сетевых адресов из документа;
- с обязательной очисткой временного каталога.

Основные параметры:

```ini
DOCOMATOR_PREVIEW_ENABLED=true
DOCOMATOR_LIBREOFFICE_BIN=/usr/bin/libreoffice
DOCOMATOR_PREVIEW_TIMEOUT_MS=120000
DOCOMATOR_PREVIEW_MAX_BYTES=134217728
```

В автономный набор пакетов включаются компоненты LibreOffice Writer и Calc. Помощник первого запуска проверяет наличие настроенного исполняемого файла.

## Проверка PDF

До фиксации результата система проверяет:

1. успешное завершение процесса;
2. наличие ровно одного PDF в целевом каталоге;
3. обычный файл вместо символической ссылки;
4. непустой допустимый размер;
5. сигнатуру `%PDF-`;
6. SHA-256 фактических байтов.

PDF хранится как отдельный неизменяемый объект. При ошибке активировать версию невозможно.

## Явная активация

Даже готовый PDF не активирует шаблон автоматически. Пользователь должен:

1. открыть предварительный просмотр;
2. отметить «Я просмотрел PDF»;
3. нажать «Активировать версию».

Активный выпуск хранит:

- пространство;
- черновик;
- кандидата и вид исходной проверенной версии;
- запрос предварительного просмотра;
- скомпилированный DOCX/XLSX;
- PDF;
- номер активного выпуска;
- неизменяемый манифест проверенных полей;
- автора, идентификатор операции и время.

Манифест версии 3 содержит полный набор полей, включая стабильный ключ, понятное название, тип, обязательность, форматтер, исходную координату и техническую привязку. Прежние манифесты версии 2 остаются неизменяемыми; добавочная миграция фиксирует для их строк явный `legacy`-формат, сохраняя прежнее отображение при повторном формировании.

Запись активного выпуска неизменяема. Отдельный текущий указатель показывает, какая версия черновика используется сейчас. Переключение указателя не изменяет прежние версии и не влияет на воспроизводимость исторических заданий.

Повторная активация того же кандидата возвращает уже существующий выпуск и восстанавливает текущий указатель без создания дубликата.

## Каталог пространства

Каталог возвращает только текущие активные выпуски выбранного пространства. Пользователь видит:

- название;
- формат;
- номер активного выпуска;
- состав: одно поле или полный набор;
- число полей;
- время активации;
- ссылку на PDF;
- ссылку на активный DOCX/XLSX.

Идентификатор версии, запроса или файла из другого пространства не раскрывает объект.

## Прикладной интерфейс

### Запросить предварительный просмотр одно-полевой версии

```http
POST /api/v1/spaces/{spaceId}/template-test-versions/{versionId}/preview
```

### Запросить предварительный просмотр многополевой версии

```http
POST /api/v1/spaces/{spaceId}/template-multi-test-versions/{versionId}/preview
```

Оба маршрута создают одну модель запроса и возвращают `versionKind`, `fieldCount`, адрес состояния и, после готовности, адрес PDF.

### Получить сохраняемое состояние

```http
GET /api/v1/spaces/{spaceId}/template-previews/{requestId}
```

### Открыть PDF

```http
GET /api/v1/spaces/{spaceId}/template-previews/{requestId}/file
```

### Активировать после просмотра

```http
POST /api/v1/spaces/{spaceId}/template-previews/{requestId}/activate
```

### Каталог активных шаблонов

```http
GET /api/v1/spaces/{spaceId}/active-templates
GET /api/v1/spaces/{spaceId}/active-templates/{activeVersionId}
GET /api/v1/spaces/{spaceId}/active-templates/{activeVersionId}/files/compiled
GET /api/v1/spaces/{spaceId}/active-templates/{activeVersionId}/files/preview
```

## Пользовательские состояния

Интерфейс объясняет:

- поставлена ли задача в очередь;
- начал ли работу LibreOffice;
- можно ли закрыть страницу;
- одно поле или полный набор выбран;
- сколько полей содержит версия;
- какая попытка выполняется;
- готов ли PDF;
- почему произошла ошибка;
- что сохранено после ошибки;
- можно ли повторить действие;
- почему кнопка активации пока недоступна;
- когда версия появилась в каталоге.

Фиктивный процент выполнения не используется.

## Отказоустойчивость

- очередь и состояние переживают перезапуск API и браузера;
- потерянная аренда задания не приводит к потере готового PDF;
- повтор обработчика видит готовое состояние и не создаёт второй объект;
- ошибка LibreOffice фиксируется отдельно от проверенной пробной версии;
- явный повтор создаёт новую попытку;
- активация повторяется идемпотентно;
- исторический активный выпуск не перезаписывается;
- один текущий указатель не допускает одновременно двух текущих выпусков одного черновика;
- прежние одно-полевые данные доступны после миграции.

## Проверки

Автоматические испытания подтверждают:

- регистрацию одно- и многополевых кандидатов в одном каталоге;
- полноту и порядок полей многополевого манифеста;
- точный перенос форматтера черновика в проверенную строку, кандидата и манифест;
- создание одного задания при повторе запроса;
- явный повтор после ошибки;
- запрет активации до готового PDF;
- отказ при неверном виде версии;
- изоляцию запросов и выпусков по пространствам;
- переключение единого текущего указателя;
- неизменяемость активного выпуска;
- скачивание активного многополевого шаблона через API;
- успешную миграцию на чистой базе и из прежней схемы.

## Ограничения текущего объёма

- визуальное сравнение двух PDF пока не выполняется;
- не поддерживаются изображения и условия; выбранный безопасный фрагмент абзаца DOCX поддерживается;
- один абзац текущего черновика может содержать только одно скалярное поле; сложные объекты и смешанное оформление внутри выбранного текста отклоняются;
- повторяющиеся таблицы/списки для `audience.members` ещё не реализованы;
- пользовательская повторяющаяся строка сводного документа пока не поддерживается.

## Следующее приращение

1. отдельный тип привязки повторяющегося блока;
2. одна повторяющаяся строка таблицы DOCX;
3. источник `audience.members` из неизменяемого снимка;
4. предел числа строк и блокирующая ошибка пустой аудитории;
5. повторяющаяся строка XLSX;
6. один сводный либо несколько индивидуальных документов.

## Связанные требования

`TPL-009`—`TPL-011`, `TPL-024`—`TPL-029`, `DOC-005`—`DOC-017`, `FLD-009`—`FLD-010`, `AUD-001`—`AUD-005`, `SPACE-005`, `SPACE-016`, `QUE-001`—`QUE-008`, `SEC-008`, `UX-004`, `UX-008`, `UX-011`—`UX-013`, `UX-030`, `OFF-013`, `AC-025`.
