# 🧱 Компилятор и пробное заполнение шаблонов

Статус: **скалярная компиляция, пробное значение и обратная проверка реализованы**  
Последнее обновление: **2026-07-12**

## Назначение

Детерминированное ядро превращает проверяемое поле черновика в техническую привязку Office Open XML, записывает пробное значение только в отдельную копию, считывает значение обратно и сохраняет проверенную версию.

```text
неизменяемый исходник
+ SHA-256 исходника
+ SHA-256 структуры
+ проверяемое поле
        ↓
повторный структурный анализ
        ↓
техническая привязка
        ↓
пробное значение
        ↓
обратное чтение
        ↓
точное сравнение
        ↓
две неизменяемые копии
```

Рабочая область:

```text
packages/template-compiler/
```

## Поддерживаемый объём

### DOCX

Поддерживается поле, связанное с целым абзацем.

Компилятор оборачивает исходный абзац в элемент управления содержимым:

```xml
<w:sdt>
  <w:sdtPr>
    <w:alias w:val="ФИО получателя"/>
    <w:tag w:val="aifield:field-id"/>
    <w:id w:val="..."/>
  </w:sdtPr>
  <w:sdtContent>
    <w:p>...</w:p>
  </w:sdtContent>
</w:sdt>
```

При пробном заполнении:

- сохраняются свойства абзаца `w:pPr`;
- сохраняются свойства первого текстового фрагмента `w:rPr`;
- старое отображаемое значение заменяется только внутри `w:sdtContent`;
- остальные части документа остаются нетронутыми;
- значение считывается обратно из технической привязки.

### XLSX

Поддерживается поле, связанное с одной ячейкой.

В `xl/workbook.xml` создаётся именованная привязка:

```xml
<definedNames>
  <definedName name="_DOCOMATOR_...">'Сотрудники'!$B$7</definedName>
</definedNames>
```

При пробном заполнении:

- сохраняется стиль ячейки;
- строка записывается как встроенная строка;
- число — как числовое значение;
- логическое значение — как `0`/`1` с русским обратным отображением «Нет»/«Да»;
- запись поверх формулы запрещена;
- значение считывается обратно из той же ячейки.

## Типы пробного значения

Поддерживаются:

| Тип | Проверка и представление |
|---|---|
| Короткая строка | до 4000 знаков |
| Длинный текст | до 20 000 знаков |
| Число | конечное число, запятая нормализуется в точку |
| Целое число | без дробной части |
| Да / нет | только логическое значение |
| Дата | строгий формат `ГГГГ-ММ-ДД` и реальная календарная дата |
| Дата и время | разбираемое значение, сохраняемое в ISO-формате |

Пустое или недопустимое пробное значение не создаёт проверенную версию.

## Проверки перед компиляцией

1. формат SHA-256;
2. фактическая SHA-256 исходника;
3. повторно построенная SHA-256 структуры;
4. существование `elementId`;
5. совпадение вида и координаты элемента;
6. отсутствие ранее созданной технической метки;
7. соответствие технической привязки сохранённому виду поля;
8. отсутствие формулы в целевой ячейке XLSX.

Браузер и локальная модель не могут передать произвольный путь части или координату без проверки по фактической структуре.

## Воспроизводимая сборка OOXML

Модуль чтения и записи ZIP:

- проверяет фактический размер каждой части;
- проверяет CRC-32;
- ограничивает число частей и суммарный объём;
- запрещает шифрование и опасные пути;
- сохраняет порядок частей;
- использует фиксированные значения времени ZIP;
- повторно рассчитывает CRC-32;
- применяет воспроизводимое сжатие;
- не переносит произвольные небезопасные атрибуты архива.

Одинаковые исходник, поле, привязка и значение дают одинаковые байты соответствующей копии.

## Обратная проверка

После компиляции:

- DOCX повторно открывается и ищется точное значение `w:tag`;
- XLSX повторно открывается и ищется точное имя `definedName`.

После пробного заполнения:

- значение считывается из `w:sdtContent` или целевой ячейки;
- считанное отображение сравнивается с нормализованным ожидаемым значением;
- при расхождении операция завершается ошибкой;
- проверенная версия не сохраняется.

## Проверенная версия

Миграция:

```text
migrations/0007_template_test_versions.sql
```

Для успешной проверки сохраняются две разные неизменяемые копии:

1. **копия с технической привязкой** — исходное отображаемое значение ещё не заменено;
2. **пробно заполненная копия** — содержит проверенное значение.

Запись хранит:

- пространство;
- черновик и поле;
- последовательный номер версии;
- SHA-256 обеих копий;
- техническую привязку;
- пробное значение;
- записанное и считанное значения;
- отчёт проверки;
- автора, время и идентификатор операции.

Запись неизменяема. Повтор той же проверки с теми же байтами и значением не создаёт дубликат.

## Прикладной интерфейс

### Выполнить пробное заполнение

```http
POST /api/v1/spaces/{spaceId}/template-drafts/{draftId}/trial
Content-Type: application/json

{
  "fieldId": "...",
  "value": "Иванов Иван Иванович"
}
```

### История проверенных версий

```http
GET /api/v1/spaces/{spaceId}/template-drafts/{draftId}/test-versions
GET /api/v1/spaces/{spaceId}/template-test-versions/{versionId}
```

### Скачать копии

```http
GET /api/v1/spaces/{spaceId}/template-test-versions/{versionId}/files/compiled
GET /api/v1/spaces/{spaceId}/template-test-versions/{versionId}/files/trial
```

Скачивание предварительно проверяет принадлежность версии пространству и фактическую SHA-256 объекта.

## Пользовательский процесс

```text
поле сохранено
      ↓
выбрать черновик и поле
      ↓
ввести пробное значение
      ↓
«Создаём техническую привязку»
      ↓
«Записываем и считываем обратно»
      ↓
✅ точное совпадение
      ↓
скачать две копии
```

При ошибке введённое значение остаётся в форме, версия не сохраняется, пользователь видит причину и идентификатор операции.

## Ограничения текущего объёма

Пока не поддерживаются:

- фрагмент внутри абзаца;
- несколько полей за один проход;
- строка или диапазон таблицы;
- повторяющиеся блоки;
- изображения;
- условия;
- полноценный предварительный просмотр LibreOffice;
- активация версии для ручного документного процесса;
- заполнение данными реальных сущностей;
- общий документ по `audience.members`.

- создание `w:sdt` и сохранение текста/оформления DOCX;
