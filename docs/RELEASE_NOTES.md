# Предварительные примечания к первому release candidate

Статус: **черновик для будущего `0.1.0-rc.1`; выпуск ещё не опубликован**.

Текущая версия исходного дерева и автономного комплекта остаётся `0.1.0-alpha.0`. Этот документ описывает уже реализованный объём и обязательные условия, после которых тот же объём может стать первым release candidate. Он не является заявлением о поддержке Debian, Astra Linux, LibreOffice или Microsoft Office без заполненной [матрицы совместимости](SUPPORT_MATRIX.md) и фактических актов.

## Назначение кандидата

Docomator — автономный сервис общего корпоративного контура без встроенной авторизации. Обычный сотрудник работает с понятными русскими названиями и не вводит UUID, SHA-256, координаты OOXML или машинные ключи.

Основной поддерживаемый путь:

1. создать сотрудников вручную либо импортировать до 1000 строк из CSV/XLSX;
2. загрузить и проверить DOCX/XLSX;
3. выбрать изменяемый текст внутри абзаца DOCX либо ячейки XLSX;
4. связать места с типизированными полями и безопасными форматтерами;
5. получить PDF-предпросмотр через установленный LibreOffice и активировать неизменяемую версию;
6. сформировать отдельный документ на каждого участника либо один сводный документ из пользовательской repeat-строки;
7. скачать DOCX/XLSX или ZIP из общего хранилища;
8. исправить пропуски и повторить только неуспешные единицы;
9. продолжить сохраняемую операцию после перезагрузки страницы или перезапуска worker.

Скачивание переводит результат в состояние «Забран», но не удаляет его. Автоматические результаты остаются в общем хранилище до явного удаления или подтверждённой очистки.

## Реализованный документный контур

- `docx.text-range` сохраняет подписи, текст до и после поля, свойства run и остальные части OOXML; прежний `docx.paragraph` остаётся совместимым.
- Декларативные форматтеры поддерживают дату, дату-время с IANA timezone, `Да`/`Нет`, русское число с 0—6 знаками и пустое необязательное значение.
- DOCX aggregate клонирует одну выбранную строку таблицы; XLSX aggregate — одну строку либо её непрерывный диапазон.
- Repeat-контракты ограничены 1000 участниками и одним блоком; стили и разрешённые свойства сохраняются, значения повторно считываются после production-render.
- Новые XLSX получают канонический `veryHidden`-лист `_AI_META`; подмена, частичный контракт и конфликт имени отклоняются.
- Сложные, неоднозначные или небезопасные OOXML-конструкции завершаются отчётом совместимости, а не попыткой молча исправить документ.

## Автономная эксплуатация

- Runtime не требует Интернет, CDN, облачных API, registry или внешнего `curl`.
- Самодостаточный bundle содержит встроенный Node.js, приложение, миграции, русскую offline UI, target gates и точный SHA-256-корпус учебных примеров.
- Preview-профиль выбирается явно. Комплект с preview обязан содержать согласованный закрытый набор LibreOffice Writer/Calc `.deb`.
- UX acceptance-профиль выбирается явно. Кандидатный комплект содержит отдельный, не устанавливаемый в production Playwright/axe-набор и Chromium из того же замкнутого `.deb`-профиля.
- Установка выполняется в неизменяемый каталог версии с атомарным переключением ссылки.
- Перед миграцией создаётся резервная копия; неуспешное обновление возвращает прежний каталог и данные.
- Ежедневный timer создаёт проверяемые копии. Восстановление проверяет manifest, типы файлов и контрольные суммы до замены данных.

## Безопасность и целостность

- LLM необязателен и не выполняет SQL, JavaScript, shell-команды, пути, OOXML или внешние действия.
- В рабочем коде запрещены `eval`, `Function`, пользовательские исполняемые выражения и произвольные dynamic imports.
- OOXML рассматривается как недоверенный ZIP/XML: проверяются пути, число частей, размеры, сжатие, внешние связи, макросы и запрещённые объекты.
- Активированные версии, готовые результаты и применённые миграции неизменяемы.
- Побочные действия используют correlation ID, idempotency key, outbox, lease и уникальные ограничения против дублей.
- SMTP и сетевые назначения разрешаются только allowlist-настройками; запись в сетевую папку требует mount, sentinel и atomic rename.

## Проверяемый baseline

Обязательный `npm run check` включает сборку, модульные и интеграционные тесты, проверку примеров, core release-gate, отдельную LibreOffice-фазу, документацию, отсутствие IAM, shell/runtime/UI и русскую терминологию.

Core gate поднимает реальные HTTP API, SQLite, worker, compiler и object storage; импортирует 10 сотрудников, выполняет intake трёх OOXML, формирует 10 личных DOCX и ZIP, пользовательские repeat-DOCX/XLSX и восстанавливает задание после аварийного завершения worker без второго результата.

LibreOffice gate сверяет SHA-256 пяти фиксированных Writer/Calc-файлов до production converter. В обычной среде без executable он явно сообщает `SKIPPED`; на preview-стенде `target-release-gate.sh` запрещает пропуск через `DOCOMATOR_REQUIRE_LIBREOFFICE=1`.

Offline `ux-acceptance-gate.sh` проверяет весь bundle и target OS profile, запускается непривилегированным пользователем, запрещает удалённый origin и выполняет точный Playwright/axe inventory без загрузок. Он сверяет установленный пакет Chromium и идентичность работающего API с bundle; отчёты связаны с Git commit, SHA-256 внутреннего manifest, SHA-256 release metadata и фактической версией браузера. Отчёты исходного дерева с меткой `development` как P5-свидетельство не принимаются.

## Известные ограничения кандидата

- Один repeat-блок на документ; вложенные и несколько независимых repeat-блоков не поддерживаются.
- Изображения, штрихкоды, сложные перекрёстные Excel-формулы и произвольный event DSL не входят в первый кандидат.
- Базовый deployment использует один сервер и один worker; broker, PostgreSQL, Kubernetes и обязательный Docker не используются.
- Локальная модель необязательна; без неё детерминированное подключение шаблона и выпуск документов должны работать.
- Учебный `examples/` детерминированно создан Docomator. Он не заменяет очищенные Office-сохранённые документы заказчика.
- Кандидатные строки Debian/Astra/LibreOffice пока имеют статус `не проверено`.
- Автономный генератор Playwright/axe-свидетельств P5 готов, но ручная доступность, Linux-визуальные эталоны и две пользовательские сессии ещё не приняты.
- Фактическое восстановление копии на отдельном целевом стенде ещё не подтверждено.

## Миграции и откат

Применённые SQL-файлы не изменяются. Добавочные миграции `0023`—`0025` сохраняют legacy-контракты ранее активированных шаблонов. Обратного SQL-перехода нет: откат выполняется проверенной предмиграционной копией вместе с предыдущим неизменяемым каталогом приложения.

Смена `0.1.0-alpha.0` на `0.1.0-rc.1` допустима только после завершения P5, успешных target gates и актов чистой установки. Обычная `0.1.0` запрещена до фактических актов Debian/Astra и успешного восстановления копии.

## Обязательные свидетельства перед публикацией RC

- два завершённых пользовательских прогона без инструкции и валидный UX-акт;
- network-free установка на чистых Debian и Astra Linux;
- обязательный target release-gate с настоящим LibreOffice без `SKIPPED`;
- Office-сохранённый очищенный корпус DOCX/XLSX с происхождением;
- проверка открытия результатов в согласованных LibreOffice и Microsoft Office;
- JSON/Markdown пилотные акты, версии ОС/glibc/Node.js/LibreOffice и SHA bundle;
- восстановление созданной копии на отдельном стенде и сравнение данных/контрольных сумм;
- отсутствие открытых блокирующих дефектов.

Текущий журнал реализации находится в `docs/CHANGELOG.md` исходного дерева, точный порядок проверки — в `docs/RELEASE_GATE.md`, а открытые внешние действия — в issue `f2re/docomator#33`.
