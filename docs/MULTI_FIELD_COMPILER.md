# 🧩 Многополевая компиляция и обратная проверка

Статус: **компиляция, проверка, повторяемая строка DOCX/XLSX, PDF-предпросмотр, активация и каталог реализованы**
Последнее обновление: **2026-07-18**

## Назначение

Подсистема обрабатывает несколько уже проверенных скалярных полей одного DOCX/XLSX за один логический проход:

```text
неизменяемый исходник
+ набор проверенных полей
        ↓
проверка исходной SHA-256 и структуры
        ↓
стабильная нормализация порядка
        ↓
последовательное создание технических привязок
        ↓
набор типизированных значений
        ↓
итоговое обратное чтение каждого поля
        ↓
неизменяемая проверенная версия
        ↓
PDF-предпросмотр и явная активация
```

## Ограничения

- от 2 до 100 полей в обычном многополевом проходе; repeat-черновик допускает от 1 до 100 полей;
- DOCX: целые абзацы и выбранные безопасные фрагменты текста;
- XLSX: отдельные ячейки;
- повторяющиеся координаты запрещены;
- повторяющиеся идентификаторы и ключи запрещены;
- поля одного запуска должны относиться к одному исходнику;
- в repeat-черновике все поля принадлежат одной выбранной строке DOCX либо выбранной строке/непрерывному диапазону XLSX; вложенность и второй repeat запрещены;
- формула XLSX не может быть полем, а сложные координатные возможности листа отклоняются до фиксации контракта.

## Воспроизводимость

Поля сортируются по стабильному ключу и идентификатору. Поэтому одинаковый набор полей даёт одинаковую техническую копию независимо от порядка элементов во входном массиве.

Перед первым изменением проверяются:

- SHA-256 исходного файла;
- SHA-256 исходной структуры;
- существование каждого структурного элемента;
- совпадение сохранённой координаты и идентификатора элемента.

Перед каждым следующим изменением структура текущей копии читается заново, а координата разрешается повторно. Это исключает доверие к устаревшему идентификатору, вычисленному до предыдущей технической вставки.

## Итоговая обратная проверка

После заполнения всех полей система повторно считывает каждое значение из окончательного файла, а не только из промежуточной копии. При расхождении ошибка содержит стабильный ключ поля.

Результат содержит:

- итоговый DOCX/XLSX;
- SHA-256 входа и выхода;
- список обработанных полей;
- список изменённых частей пакета;
- записанное и считанное значение каждого поля;
- количество проверенных привязок и значений.

## Проверенная многополевая версия

Добавляемая схема хранит техническую и проверенную копии, полный набор значений и неизменяемую строку каждого поля. Существующие одно-полевые версии сохраняются для совместимости.

Пользовательский маршрут `trial-all` требует значение для каждого поля черновика. Для repeat-черновика он компилирует внешний `airepeat:*` DOCX либо `_DOCOMATOR_REPEAT_*` XLSX, проверяет хотя бы одну пробную строку и замораживает полный форматно-согласованный repeat-контракт вместе с многополевой версией и кандидатом. После успеха доступны история, сведения обратного чтения и скачивание обеих копий. Повтор того же полного набора не создаёт дубликат.

## Выпуск, предварительный просмотр и активация

Одно- и многополевые проверенные версии регистрируются как единые кандидаты на выпуск. Для каждого кандидата фиксируются:

- вид исходной проверенной версии;
- пространство и черновик;
- номер исходной версии;
- техническая и пробно заполненная копии;
- число и упорядоченный набор полей;
- исходные и технические привязки каждого поля.

Дальнейший процесс одинаков для обоих видов версии:

```text
кандидат на выпуск
→ сохраняемое задание `template.preview`
→ ограниченный LibreOffice
→ проверенный PDF
→ явное подтверждение пользователя
→ неизменяемый активный выпуск
→ единый текущий указатель каталога
```

Активный манифест включает весь набор полей, их типы, обязательность, форматтеры, исходные координаты, технические привязки, уровень совместимости и массив повторов. DOCX и legacy-XLSX используют версию 4, а новые XLSX с `_AI_META` — версию 5 и секцию `xlsxMetadata`. Старые активные версии остаются доступны по идентификатору, а текущий указатель изменяется отдельно.

Миграция переносит прежние одно-полевые предварительные просмотры, активные версии и указатели в единый слой выпусков без изменения исходных записей.

## Проверки

Эталонные испытания подтверждают:

- одинаковый результат для разного порядка входных полей;
- две независимые привязки DOCX;
- две независимые привязки XLSX;
- точный общий реестр полей XLSX на `veryHidden`-листе `_AI_META`;
- сохранение соседнего текста и ячейки;
- типизированное заполнение строки и целого числа;
- итоговое обратное чтение всех полей;
- отказ при повторяющейся координате;
- отказ при устаревшем идентификаторе элемента;
- один каталог для одно- и многополевых выпусков;
- устранение повторов запроса PDF и активации;
- изоляцию предварительного просмотра и выпуска по пространствам;
- полноту многополевого активного манифеста;
- скачивание многополевого активного шаблона через API;
- сохранение и строгую проверку XLSX repeat-контракта в SQLite;
- production-render XLSX на 10 участников через настоящий API, очередь и worker.
