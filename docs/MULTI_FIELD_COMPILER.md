# 🧩 Многополевая компиляция и обратная проверка

Статус: **детерминированное ядро реализовано; хранение и активация многополевой версии относятся к следующему приращению**  
Последнее обновление: **2026-07-13**

## Назначение

Подсистема обрабатывает несколько уже проверенных скалярных полей одного DOCX/XLSX за один логический проход:

```text
неизменяемый исходник
+ набор проверенных полей
        ↓
проверка исходной SHA-256 и структуры
        ↓
стабильная нормализация порядка
        ↓
последовательное создание технических привязок
        ↓
набор типизированных значений
        ↓
итоговое обратное чтение каждого поля
```

## Ограничения

- от 1 до 100 полей;
- DOCX: целые абзацы;
- XLSX: отдельные ячейки;
- повторяющиеся координаты запрещены;
- повторяющиеся идентификаторы и ключи запрещены;
- поля одного запуска должны относиться к одному исходнику;
- повторяющиеся таблицы и `audience.members` пока не входят в этот подэтап.

## Воспроизводимость

Поля сортируются по стабильному ключу и идентификатору. Поэтому одинаковый набор полей даёт одинаковую техническую копию независимо от порядка элементов во входном массиве.

Перед первым изменением проверяются:

- SHA-256 исходного файла;
- SHA-256 исходной структуры;
- существование каждого структурного элемента;
- совпадение сохранённой координаты и идентификатора элемента.

Перед каждым следующим изменением структура текущей копии читается заново, а координата разрешается повторно. Это исключает доверие к устаревшему идентификатору, вычисленному до предыдущей технической вставки.

## Итоговая обратная проверка

После заполнения всех полей система повторно считывает каждое значение из окончательного файла, а не только из промежуточной копии. При расхождении ошибка содержит стабильный ключ поля.

Результат содержит:

- итоговый DOCX/XLSX;
- SHA-256 входа и выхода;
- список обработанных полей;
- список изменённых частей пакета;
- записанное и считанное значение каждого поля;
- количество проверенных привязок и значений.

## Проверки

Эталонные испытания подтверждают:

- одинаковый результат для разного порядка входных полей;
- две независимые привязки DOCX;
- две независимые привязки XLSX;
- сохранение соседнего текста и ячейки;
- типизированное заполнение строки и целого числа;
- итоговое обратное чтение всех полей;
- отказ при повторяющейся координате;
- отказ при устаревшем идентификаторе элемента.

## Следующий вертикальный шаг

1. сохранить многополевую проверенную версию;
2. сформировать единый отчёт пробных значений;
3. создать PDF-предпросмотр этой версии;
4. активировать многополевой манифест;
5. затем добавить повторяющуюся строку/таблицу для `audience.members`.
