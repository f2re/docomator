# Ближайшие приращения Docomator

Статус: **многополевая компиляция, проверка, хранение и интерфейс реализованы; следующий шаг — многополевая активация и повторяющийся блок аудитории**
Последнее обновление: **2026-07-13**

План подчиняется [нормативным требованиям](REQUIREMENTS.md), [архитектуре](ARCHITECTURE.md) и общему [плану реализации](IMPLEMENTATION_PLAN.md). Каждое приращение должно оставлять систему запускаемой, автономной и пригодной для проверки без локальной модели.

## Ограничения, действующие для всех шагов

- содержимое DOCX/XLSX считается недоверенным;
- исходный XML, пути и команды из документа не исполняются;
- локальная модель не получает права изменять файлы, БД или внешние системы;
- данные, черновики, предварительные просмотры и активные версии не смешиваются между пространствами;
- каждое длительное действие имеет понятное русское состояние и следующий шаг;
- исходный файл, проверенная и активная версии шаблона неизменяемы;
- новые действия с побочным эффектом получают идентификатор операции и защиту от повторов;
- интерфейс работает без внешних сетевых ресурсов и не требует знания английских терминов;
- новый шаг не считается завершённым без сборки, тестов, документации и проверки автономной поставки.

## Состояние ближайшей цепочки

| Шаг | Состояние | Проверяемый результат |
|---|---:|---|
| 1. Безопасная проверка пакета | ✅ | повреждённый, макросный или опасный файл отклоняется до сохранения |
| 2. Неизменяемый карантин | ✅ | принятый исходник сохраняется по SHA-256 внутри одного пространства |
| 3. Структурное представление | ✅ | абзацы DOCX и ячейки XLSX имеют устойчивые координаты и ограниченный ответ |
| 4. Сохранение поля | ✅ | сервер повторно проверяет исходник и сам получает координату элемента |
| 5. Скалярная компиляция | ✅ | DOCX получает `w:sdt`, XLSX — именованную привязку |
| 6. Пробное заполнение | ✅ | значение вставляется, считывается обратно и проверяется |
| 7. Проверенная версия | ✅ | скомпилированная и пробная копии сохраняются неизменяемо с SHA-256 |
| 8. Предварительный просмотр | ✅ | сохраняемое задание создаёт проверенный PDF либо понятную ошибку |
| 9. Активация версии | ✅ | после явного подтверждения версия появляется в каталоге пространства |
| 10. Несколько полей | 🟡 | компиляция, полный набор значений, хранение и скачивание работают; PDF/активация впереди |
| 11. Повторяющийся блок | ⬜ | `audience.members` выводится таблицей или списком в одном общем документе |
| 12. Документные задания | ⬜ | снимок аудитории создаёт один сводный либо несколько индивидуальных документов |

## Приращение A — структурное представление

**Состояние:** ✅ завершено и проверено.

Реализованы безопасный разбор DOCX/XLSX, устойчивые структурные идентификаторы, ограничения ответа и русскоязычный выбор элемента без передачи исходного XML браузеру.

## Приращение B — поле и проверяемая привязка

**Состояние:** ✅ завершено и проверено.

Реализованы черновики пространства, проверяемые поля, повторное чтение исходника по SHA-256, серверная проверка координаты, защита от повторных ключей и межпространственных ссылок.

## Приращение C — базовый компилятор

**Состояние:** ✅ завершено и проверено.

Реализованы `w:sdt` для целого абзаца DOCX, именованная привязка одной ячейки XLSX, воспроизводимая сборка пакета и обратное обнаружение технической привязки.

## Приращение D — пробное заполнение и проверенная версия

**Состояние:** ✅ завершено и проверено.

Реализованы типизированное пробное значение, сохранение форматирования, запрет записи поверх формулы XLSX, точное обратное чтение, две неизменяемые копии и история проверенных версий.

## Приращение E — предварительный просмотр и активация

**Состояние:** ✅ завершено и проверено.

Реализовано:

1. сохраняемое задание `template.preview`;
2. отдельный временный профиль LibreOffice;
3. запуск без оболочки, внешний тайм-аут и принудительное завершение процесса;
4. ограничение размера входа, журнала процесса и PDF;
5. проверка сигнатуры PDF и хранение по SHA-256;
6. состояния ожидания, выполнения, готовности и ошибки;
7. явный повтор после неуспешной попытки;
8. встроенный просмотр PDF и ссылка для отдельного открытия;
9. обязательная отметка пользователя «Я просмотрел PDF»;
10. запрет активации без готового PDF;
11. неизменяемая активная версия и отдельный текущий указатель;
12. каталог активных шаблонов одного пространства;
13. скачивание активного шаблона и PDF;
14. события, аудит, идентификатор операции и межпространственная изоляция.

Критерий завершения выполнен: проверенная версия проходит сохраняемое создание PDF, пользователь явно подтверждает результат, после чего версия становится текущей в каталоге пространства без изменения прежних версий.

## Приращение F — несколько полей в одной версии

**Состояние:** 🟡 компиляция и проверенная версия завершены; активация выполняется следующим подэтапом.

Реализовано:

1. упорядоченная компиляция от 2 до 100 полей;
2. запрет повторяющихся идентификаторов, ключей и координат;
3. повторное разрешение координаты перед каждой вставкой;
4. типизированный полный набор значений;
5. итоговое обратное чтение каждого значения из окончательного файла;
6. единый отчёт и ошибка с ключом поля;
7. совместимая добавляемая схема многополевых проверенных версий;
8. дочерние неизменяемые строки каждого поля;
9. устранение повторов одинакового полного набора;
10. API, форма «Проверить все поля», история и скачивание двух копий;
11. изоляция по пространствам, события и аудит.

Остаётся:

- передать многополевую проверенную версию в сохраняемое создание PDF;
- активировать манифест со всеми полями;
- сохранить прежнюю активную версию для исторических заданий.

Критерий полного приращения: один DOCX/XLSX с несколькими полями воспроизводимо компилируется, заполняется, проверяется, просматривается и активируется.

## Приращение G — общий документ по аудитории

**Цель:** поддержать не только документ на каждого человека, но и один сводный документ.

Входит:

1. привязка повторяющейся строки, абзаца или диапазона;
2. источник коллекции `audience.members`;
3. устойчивый порядок участников из неизменяемого снимка;
4. таблица DOCX, список DOCX и повторяющаяся строка XLSX;
5. пустая аудитория как блокирующая ошибка;
6. предельное число строк и понятный отчёт;
7. тесты общего документа и режима «по документу на каждого» на одном наборе участников.

Критерий завершения: одна и та же аудитория воспроизводимо создаёт либо `N` индивидуальных файлов, либо один файл с `N` строками/элементами.

## После детерминированного пути

Только после нескольких полей, повторяющегося блока и рабочего документного задания подключаются локальные агенты:

- поиск вероятных полей;
- сопоставление с существующими свойствами;
- предложение формата ФИО и даты;
- группировка уточняющих вопросов;
- создание разрешённых текстовых блоков.

Любое предложение агента остаётся данными, проходит схему и подтверждение, а файловые изменения выполняет только компилятор.
