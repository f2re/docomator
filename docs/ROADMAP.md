# План развития Docomator

План развития отражает фактическое состояние реализации, но не заменяет [нормативные требования](REQUIREMENTS.md). Подробная последовательность ближайших шагов находится в [NEXT_ITERATIONS.md](NEXT_ITERATIONS.md).

## Обозначения

- ✅ завершено и проверено;
- 🟡 выполняется или доступен базовый объём;
- ⬜ не начато;
- ⛔ заблокировано решением или внешней зависимостью.

## Ближайший приоритет

Следующие вертикальные приращения выполняются без локальной модели:

1. сохранить поле: название, ключ, тип, обязательность, идентификатор элемента и `structure_sha256`;
2. проверить принадлежность черновика пространству и существование координаты;
3. создать техническую привязку DOCX/XLSX без изменения исходника;
4. выполнить пробное скалярное заполнение и обратное чтение;
5. сохранить неизменяемую версию шаблона;
6. добавить повторяющийся блок для `audience.members`;
7. создать один сводный либо несколько индивидуальных документов из одного снимка аудитории;
8. только затем подключить агента поиска кандидатов полей.
