# Сквозной release-gate

Статус: **обязательный core gate реализован; реальный LibreOffice проверяется отдельной фазой**
Последнее обновление: **2026-07-19**

## Назначение

Release-gate подтверждает, что собранные runtime-процессы совместно формируют документы на свежей базе, а не только проходят изолированные модульные тесты. Он входит в `npm run check` и не использует сеть, SMTP, сетевую папку, LLM или внешний broker.

Связанный срез требований: `TPL-009`, `TPL-024`—`TPL-030`, `DOC-005`—`DOC-019`, `QUE-001`—`QUE-005`, `QUE-008`, `OFF-010`, `OFF-013`, `NFR-001`, `NFR-007`, `NFR-009`; критерии `AC-003`, `AC-010`, `AC-018`, `AC-024`—`AC-026`.

## Обязательный core gate

Команда:

```bash
npm run build
npm run test:release-gate
```

Оркестратор `scripts/ci/release-gate.mjs`:

1. создаёт закрытый временный каталог с sentinel и применяет миграции 0001—0025;
2. реальным Document IR и compiler собирает text-range DOCX, пользовательский repeat-DOCX и repeat-XLSX;
3. сохраняет исходники, версии, кандидаты и активные выпуски в настоящих SQLite и content-addressed object store;
4. запускает настоящий Fastify API отдельным процессом на динамическом loopback-порту, полученном через IPC;
5. через HTTP создаёт снимки, выполняет preflight, создаёт задания и проверяет идемпотентный повтор;
6. запускает crash-worker на production queue/handler/registry, ждёт сохранения первой единицы и аварийно завершает его `SIGKILL`;
7. после фактического истечения сохранённой аренды запускает обычный `apps/worker/dist/main.js` и проверяет восстановление без второго результата;
8. скачивает ZIP из 10 персональных DOCX, побайтно сверяет его части с individual outputs и выполняет reverse-read каждого OOXML;
9. после настоящего перезапуска worker формирует пользовательский repeat-DOCX на 10 участников;
10. отдельным production-заданием формирует пользовательский repeat-XLSX на тех же 10 участников;
11. для DOCX проверяет шапку, подпись, порядок и число строк, уникальные Word-ID, отсутствие плейсхолдеров и стандартизированного fallback;
12. для XLSX проверяет порядок и число строк, стили и высоту, перевод формул, `dimension`, отсутствие плейсхолдеров, технических defined names и стандартизированного fallback, а также точный неизменённый реестр field/repeat на `veryHidden`-листе `_AI_META`;
13. подтверждает через read-only SQL ровно 10 персональных единиц, 10 уникальных outputs, восстановленную аренду и по одному итоговому dedupe-событию каждого сводного задания.

Core gate использует уже проверенный PDF-объект с явной меткой `release-gate-stub` только для создания активного выпуска. Так renderer и восстановление worker не зависят от установленного LibreOffice. Сам LibreOffice не подменяется молча и проверяется следующей отдельной фазой.

## Gate реального LibreOffice

```bash
npm run test:release-gate:libreoffice
```

Если локальный executable найден, `scripts/ci/libreoffice-release-gate.mjs` преобразует настоящие DOCX и XLSX через production `convertOfficeToPdf`, проверяет `%PDF-`, ограниченный размер, metadata преобразователя, отдельный профиль и очистку временного каталога каждого преобразования.

Если LibreOffice отсутствует, команда явно печатает `SKIPPED`. На эталонном Debian/Astra-стенде отсутствие превращается в ошибку:

```bash
DOCOMATOR_REQUIRE_LIBREOFFICE=1 \
DOCOMATOR_LIBREOFFICE_BIN=/usr/bin/libreoffice \
npm run test:release-gate:libreoffice
```

Версия executable и ОС фиксируются в акте пилота; эта стендовая обязанность не заменяется CI-пропуском.

## Запуск из автономного комплекта

Bundle содержит production-pruned зависимости, оба gate и crash-worker. На целевом стенде не требуется исходный репозиторий, `npm ci` или сеть:

```bash
"$BUNDLE_ROOT/target-release-gate.sh" \
  --config /etc/docomator/docomator.env
```

Сценарий сначала повторно проверяет полный bundle, требует точного совпадения target ОС/версии/архитектуры с package profile и совпадения preview-профиля target config с bundle. Затем он запускает core gate встроенным Node.js и, если preview включён, принудительно задаёт `DOCOMATOR_REQUIRE_LIBREOFFICE=1`. Поэтому отсутствие converter не превращается в `SKIPPED`. Профиль `--without-preview` явно сообщает, что LibreOffice gate к нему неприменим и не может служить актом поддержки preview.

## Безопасность и очистка

- дочерние процессы получают whitelist окружения и явные выключатели SMTP, LLM и preview;
- API слушает только `127.0.0.1` и получает порт `0`, поэтому gate не обращается к заранее существующему сервису;
- процессы завершаются только по сохранённым PID; поиск и `pkill` не используются;
- stdout/stderr ограничены кольцевым буфером и печатаются при ошибке;
- ZIP проверяется по числу, именам и expanded size частей;
- временный каталог удаляется только после `realpath`, проверки родителя, имени и sentinel;
- `DOCOMATOR_RELEASE_GATE_KEEP=1` сохраняет каталог для локальной диагностики.
