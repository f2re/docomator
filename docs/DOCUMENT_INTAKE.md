# 🛡️ Безопасный приём, карантин и структура DOCX/XLSX

Статус: **проверка, неизменяемый карантин, структура и ручной выбор полей/repeat-области реализованы**
Последнее обновление: **2026-07-18**

## Назначение

Подсистема защищает систему до подключения документа как шаблона и отвечает на пять вопросов:

1. можно ли безопасно прочитать пакет;
2. какие возможности Office требуют внимания;
3. можно ли сохранить исходник;
4. какие абзацы, текстовые фрагменты, листы и ячейки доступны для разметки;
5. какой следующий шаг допустим.

Проверка и структурный анализ выполняются локально, без локальной модели и без запуска содержимого документа.

## Пользовательский процесс

```text
выбор DOCX/XLSX
      ↓
проверка имени, размера и ZIP
      ↓
поиск опасных частей и внешних связей
      ↓
русскоязычный отчёт
      ↓
явное подтверждение пользователя
      ↓
повторная проверка и SHA-256
      ↓
неизменяемый карантин пространства
      ↓
структурный анализ
      ↓
выбор абзаца или ячейки
```

Проверка, сохранение и структурный анализ являются разными действиями. Файл не сохраняется только потому, что пользователь выбрал его или получил отчёт.

## Возможные результаты проверки

| Состояние | Значение |
|---|---|
| **Структура прошла проверку** | Блокирующие особенности не найдены. Исходник можно сохранить и передать структурному анализу. |
| **Файл принят с замечаниями** | Сохранение возможно после просмотра предупреждений; пробное формирование будет обязательным. |
| **Файл нельзя использовать** | Найдена блокирующая или повреждённая структура. Сохранение и структурный анализ запрещены. |

## Реализованные проверки

- расширение и безопасное имя файла;
- сигнатура ZIP;
- максимальный размер исходного файла;
- максимальное число частей;
- суммарный распакованный размер;
- размер отдельной части;
- подозрительно высокая степень сжатия;
- абсолютные пути и выход через `..`;
- обратные косые черты и управляющие символы;
- повторяющиеся и различающиеся только регистром имена частей;
- зашифрованные части;
- символические ссылки;
- неподдерживаемый способ сжатия;
- обязательные части DOCX/XLSX;
- макросы VBA;
- ActiveX;
- встроенные OLE-объекты;
- цифровые подписи;
- внешние связи и внешние книги;
- сложные схемы и пользовательские XML-данные;
- запрещённые объявления `DOCTYPE` и `ENTITY` до структурного разбора.

## Ограничения по умолчанию

| Ограничение | Значение |
|---|---:|
| Размер исходного файла | 32 МБ |
| Число частей | 2048 |
| Суммарный распакованный объём | 256 МБ |
| Размер отдельной части | 32 МБ |
| Размер служебной XML-части при первичной проверке | 4 МБ |
| Размер структурной XML-части | 16 МБ |
| Максимальная степень сжатия | 200:1 |

Ограничения меняются только вместе с нагрузочными и отрицательными проверками.

## Неизменяемый карантин

После явного подтверждения сервер повторно выполняет проверку. Если решение остаётся допустимым:

1. вычисляется SHA-256 фактических байтов;
2. файл сохраняется в адресуемом по содержимому хранилище;
3. создаётся неизменяемая запись карантина;
4. сохраняется полный отчёт проверки;
5. фиксируются пространство, автор, время и идентификатор операции;
6. записываются событие предметной области и аудит.

Запись карантина нельзя изменить или удалить обычной прикладной операцией. Повторная загрузка того же файла:

- не создаёт второй физический объект;
- не создаёт дубликат записи в том же пространстве;
- может создать отдельную запись в другом пространстве без смешивания метаданных.

Путь файла не строится из пользовательского имени. Физический объект хранится по SHA-256.

## Структурное представление

### DOCX

Извлекаются:

- основной текст `word/document.xml`;
- верхние и нижние колонтитулы;
- сноски и концевые сноски;
- абзацы;
- текстовые фрагменты;
- признаки жирного и курсивного начертания;
- табуляция и переносы строки;
- положение абзаца внутри таблицы: номер таблицы, строки и ячейки.

### XLSX

Извлекаются:

- книга и её внутренние связи;
- названия листов и пути их частей;
- адреса ячеек;
- общие строки и встроенные строки;
- числа, логические значения и ошибки;
- формулы и последнее сохранённое значение формулы.

### Устойчивые координаты

Идентификатор элемента вычисляется из:

```text
SHA-256 исходного файла
+ путь внутренней части
+ структурная координата
```

Повторный анализ неизменяемого файла возвращает те же идентификаторы. Дополнительно рассчитывается SHA-256 структурного представления.

Крупный документ возвращает ограниченное число элементов, но сохраняет полные итоговые количества. Пользователь явно видит, что показана ограниченная выборка.

Исходный XML не возвращается браузеру как исполняемое содержимое.

## Прикладной интерфейс

### Первичная проверка

```http
POST /api/v1/document-intake/inspect?fileName=Шаблон.docx
Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
```

### Структурный анализ

```http
POST /api/v1/document-intake/analyze?fileName=Шаблон.docx&limit=300
Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
```

Ответ содержит:

```json

      "partsRead": 1,
      "paragraphs": 12,
      "runs": 18,
      "sheets": 0,
      "cells": 0,
      "formulas": 0,

        "index": 0,
```

### Подтверждённое сохранение

```http
POST /api/v1/spaces/{spaceId}/document-sources/quarantine?fileName=Шаблон.docx
Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
```

Сервер не доверяет предыдущему клиентскому отчёту и снова проверяет переданный файл.

### Список сохранённых исходников

```http
GET /api/v1/spaces/{spaceId}/document-sources?limit=50
GET /api/v1/spaces/{spaceId}/document-sources/{recordId}
```

Идентификатор записи из другого пространства возвращается как недоступный ресурс.

## Граница безопасности

На текущем этапе файл:

- не выполняется;
- не передаётся локальной модели;
- не распаковывается в пользовательский каталог;
- не становится активным шаблоном;
- не вызывает внешние связи;
- не запускает LibreOffice;
- сохраняется только после явного подтверждения;
- сохраняется только при допустимом решении проверки;
- не передаёт исходный XML пользовательскому интерфейсу.

## Следующие изменения

### Сохранение поля

- выбор абзаца, текстового фрагмента, ячейки, строки или диапазона;
- название, стабильный ключ, тип и обязательность;
- сохранение `source_sha256`, `structure_sha256`, идентификатора элемента и координаты;
- серверная проверка существования элемента;
- повторное открытие разметки без потери координат.

### Компиляция и пробное заполнение

- элемент управления содержимым DOCX;
- именованная привязка XLSX;
- изменение только разрешённой части пакета;
- пробное значение и обратное чтение;
- предварительный просмотр LibreOffice;
- запрет активации при ошибке.

### Сравнение примеров и повторяющийся блок

- сравнение пустого шаблона с заполненным экземпляром;
- сравнение нескольких заполненных экземпляров;
- детерминированные кандидаты изменяемых областей;
- автоматическое предложение таблицы или списка для `audience.members`.

Ручной путь позволяет выбрать одну строку таблицы DOCX либо всю использованную строку/непрерывный диапазон одной строки XLSX и связать её с `audience.members`. Клиент передаёт только стабильные идентификаторы элементов, а координату, границы и источник выводит и повторно проверяет backend по сохранённой структуре. При усечённой XLSX-структуре вся строка недоступна; формула не может быть полем. Автоматическое обнаружение кандидата по примерам, списки и несколько repeat-областей пока не реализованы.

Поиск полей локальной моделью подключается только после работоспособного ручного процесса.
